## 원-핫 인코딩
원 핫 인코딩은 어휘집의 모든 단어가 n개 일때 각 단어를 n차원 벡터로 변환한다
각 단어마다 고유한 정수를 부여하고 이 벡터에서 해당 단어에게 부여된 정수의 위치(index)만 1로, 나머지는 0으로 채운다
이렇게 얻은 백터를 원-핫 벡터(one-hot vector)라고 한다
### 원-핫 벡터(One-hot Vector)
$$v_i = [x_1, x_2, \dots, x_{|V|}]^T, \quad x_j = \begin{cases} 1 & \text{if } j = i \\ 0 & \text{if } j \neq i \end{cases}$$
- **① 수식 전체 의미**: 특정 단어 $i$를 전체 사전 크기 $|V|$ 차원의 공간에서 유일하게 1의 값을 가진 축으로 표현합니다.
    
- **② 각 기호의 의미**:
	- $v_i$: $i$번째 단어의 원-핫 벡터        
    - $|V|$: 전체 어휘 사전의 크기(차원 수)        
    - $x_j$: 벡터의 $j$번째 요소 값        
    - $T$: 전치(Transpose), 열벡터를 행벡터 형태로 가독성 있게 표기
        
- **③ 실제 계산 방식**:
	- **어휘 사전 구축**: `{ 'a': 0, 'am': 1, 'hello': 2, 'i': 3, 'student': 4 }` 
		- (알파벳 순 정렬 가정)
	- **단어별 벡터 생성**:
         - `hello` = $[0, 0, 1, 0, 0]^T$
         - `i` = $[0, 0, 0, 1, 0]^T$
         - `am` = $[0, 1, 0, 0, 0]^T$
         - `a` = $[1, 0, 0, 0, 0]^T$
        - `student` = $[0, 0, 0, 0, 1]^T$
            
- **④ 왜 필요한가**: 단어 간의 유클리드 거리(Euclidean Distance)를 모두 $\sqrt{2}$로 동일하게 만들어, 초기 상태에서 모델이 특정 단어들이 수학적으로 더 가깝다고 편향되게 학습하는 것을 방지합니다.
