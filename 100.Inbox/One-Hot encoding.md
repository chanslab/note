# 원-핫 인코딩(One-Hot Encoding)

## 1. 개념

범주형 데이터(Categorical Data)를 수치형 벡터(Numeric Vector)로 변환하는 인코딩(Encoding) 방식이다. 해당 범주에 해당하는 위치만 1, 나머지는 모두 0으로 표현한다.

**예시: 색상(Color) → 원-핫 벡터(One-Hot Vector)**

| 범주        | 벡터 표현     |
| --------- | --------- |
| 빨강(Red)   | [1, 0, 0] |
| 초록(Green) | [0, 1, 0] |
| 파랑(Blue)  | [0, 0, 1] |

---

## 2. 필요성

모델(Model)은 수치 연산만 가능하다. 범주형 데이터를 그대로 입력하면 연산이 불가능하다.

**단순 정수 인코딩(Integer Encoding)의 문제점**

색상을 빨강=1, 초록=2, 파랑=3으로 인코딩하면, 모델은 **파랑(3) > 초록(2) > 빨강(1)** 이라는 순서 관계(Ordinal Relationship)가 존재한다고 잘못 학습한다. 원-핫 인코딩은 각 범주를 동등한 거리(Equidistant)로 표현하여 이 문제를 방지한다.

---

## 3. 동작 원리

범주(Category)의 수를 $N$이라 할 때, 크기 $N$의 벡터를 생성하고 해당 인덱스(Index)만 1로 설정한다.

$$\mathbf{e}_i = [0, \ldots, 0, \underbrace{1}_{i\text{번째}}, 0, \ldots, 0] \in {0,1}^N$$

**수식 설명**

① **수식 전체 의미** $N$개의 범주 중 $i$번째 범주를 $N$차원 이진 벡터(Binary Vector)로 표현한다.

② **기호 정의**

|기호|명칭|의미|
|---|---|---|
|$\mathbf{e}_i$|표준 기저 벡터(Standard Basis Vector)|$i$번째 범주를 나타내는 벡터|
|$N$|범주 수(Number of Categories)|전체 고유 범주의 개수|
|$i$|인덱스(Index)|해당 범주의 위치 (0 또는 1부터 시작)|
|${0,1}^N$|N차원 이진 공간|각 원소가 0 또는 1인 N차원 벡터 공간|

③ **계산 예시** 범주: {개, 고양이, 새}, $N=3$

- 개 → $i=0$ → $[1, 0, 0]$
- 고양이 → $i=1$ → $[0, 1, 0]$
- 새 → $i=2$ → $[0, 0, 1]$

④ **존재 이유** 범주 간 크기 관계나 순서 관계를 수치에 반영하지 않고, 각 범주를 완전히 독립적인 축(Axis)으로 표현하기 위해 필요하다.

---

## 4. 인코딩 방식 비교

|항목|정수 인코딩(Integer Encoding)|원-핫 인코딩(One-Hot Encoding)|
|---|---|---|
|표현 방식|범주를 단일 정수로 매핑|범주를 이진 벡터로 매핑|
|순서 관계|암묵적 순서 발생|순서 없음|
|벡터 크기|1|$N$ (범주 수만큼 증가)|
|적합한 데이터|순서형(Ordinal) 범주|명목형(Nominal) 범주|
|메모리 사용|적음|범주 수 증가 시 급증|
|희소성 문제|없음|고차원 희소 벡터(Sparse Vector) 발생|

---

## 5. 활용처

|분야|사용 예시|
|---|---|
|머신러닝 전처리(ML Preprocessing)|선형 회귀(Linear Regression), 랜덤 포레스트(Random Forest) 입력|
|자연어 처리(NLP)|단어 집합(Vocabulary)의 각 단어를 벡터로 표현|
|신경망 출력층(Output Layer)|다중 클래스 분류(Multi-class Classification)의 레이블(Label) 표현|
|손실함수 연동|크로스 엔트로피(Cross-Entropy) 계산 시 타깃(Target) 벡터로 사용|

---

## 6. 한계 및 대안

**한계**

- 범주 수 $N$이 크면 벡터 차원이 폭발적으로 증가한다 → 차원의 저주(Curse of Dimensionality)
- 벡터 간 의미적 유사도(Semantic Similarity)를 표현하지 못한다 (개와 고양이가 동일한 거리)

**대안**

|문제|대안 기법|
|---|---|
|고차원 희소 문제|임베딩(Embedding) — 밀집 저차원 벡터로 압축|
|의미 관계 미반영|워드 임베딩(Word Embedding, Word2Vec 등)|
|순서형 범주|레이블 인코딩(Label Encoding) 유지|
