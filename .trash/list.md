파이썬 list의 주요 특성은 다음과 같습니다.

1. 순서가 있다

입력한 순서대로 저장됩니다.

인덱스(index)로 접근할 수 있습니다.


a = [10, 20, 30]
print(a[0])   # 10
print(a[1])   # 20


2. 변경 가능하다 (mutable)

값을 추가, 수정, 삭제할 수 있습니다.


a = [1, 2, 3]
a[1] = 200
a.append(4)
print(a)   # [1, 200, 3, 4]


3. 여러 자료형을 함께 저장할 수 있다

숫자, 문자열, 불리언 등을 한 리스트 안에 넣을 수 있습니다.


a = [1, "hello", True, 3.14]


4. 중복을 허용한다

같은 값이 여러 번 들어갈 수 있습니다.


a = [1, 1, 2, 2, 3]


5. 크기가 고정되지 않는다

필요에 따라 원소를 계속 추가하거나 제거할 수 있습니다.


a = []
a.append(10)
a.append(20)


6. 슬라이싱이 가능하다

일부 구간만 잘라서 사용할 수 있습니다.


a = [10, 20, 30, 40, 50]
print(a[1:4])   # [20, 30, 40]


7. 반복문과 함께 자주 사용된다

a = [10, 20, 30]
for x in a:
    print(x)


8. 다른 리스트를 원소로 가질 수 있다

즉, 중첩 리스트가 가능합니다.


a = [[1, 2], [3, 4], [5, 6]]
print(a[0][1])   # 2


9. 대표 메서드가 많다

append(): 맨 뒤에 추가

insert(): 특정 위치에 삽입

remove(): 값 삭제

pop(): 인덱스로 꺼내며 삭제

sort(): 정렬

reverse(): 뒤집기




예:

a = [3, 1, 2]
a.sort()
print(a)   # [1, 2, 3]

10. 튜플(tuple)과 차이가 있다



list: 변경 가능

tuple: 변경 불가


예:

a = [1, 2, 3]   # list
b = (1, 2, 3)   # tuple

한 줄로 정리하면,
파이썬 list는 순서가 있고, 변경 가능하며, 중복을 허용하는 여러 값을 저장하는 자료형입니다.

아래처럼 정리하면 가장 이해하기 쉽습니다.

1. 한눈에 보는 차이

자료형	기호	순서	변경 가능	중복 허용	인덱스 접근	특징

list	[ ]	있음	가능	허용	가능	가장 일반적인 묶음형
tuple	( )	있음	불가능	허용	가능	읽기 전용 성격
set	{ }	없음	가능	불가	불가	중복 제거에 유용
dict	{key: value}	있음*	가능	키는 중복 불가	키로 접근	key-value 구조


* dict는 파이썬 3.7+에서 입력 순서를 유지합니다.

2. 각각의 특징

list

여러 값을 순서대로 저장하고, 나중에 수정 가능한 자료형입니다.

a = [10, 20, 30]
a[1] = 200
a.append(40)

print(a)   # [10, 200, 30, 40]

언제 쓰나:

값을 순서대로 저장해야 할 때

추가, 삭제, 수정이 자주 있을 때



---

tuple

list와 비슷하지만 한 번 만들면 바꿀 수 없는 자료형입니다.

a = (10, 20, 30)
print(a[0])   # 10

아래는 오류가 납니다.

a[1] = 200   # 오류

언제 쓰나:

값이 바뀌면 안 될 때

고정된 데이터 묶음을 표현할 때

함수가 여러 값을 반환할 때 자주 사용


point = (3, 5)


---

set

중복을 허용하지 않는 자료형입니다.
순서가 없고, 인덱스로 접근할 수 없습니다.

a = {1, 2, 2, 3, 3, 3}
print(a)   # {1, 2, 3}

언제 쓰나:

중복 제거할 때

포함 여부를 빠르게 확인할 때


a = {1, 2, 3}
print(2 in a)   # True


---

dict

key와 value를 짝으로 저장하는 자료형입니다.

person = {
    "name": "Kim",
    "age": 30,
    "job": "developer"
}

print(person["name"])   # Kim

언제 쓰나:

이름표를 붙여서 데이터를 관리할 때

속성 정보를 저장할 때


예:

student = {
    "학번": "2026001",
    "이름": "홍길동",
    "점수": 95
}

3. 같은 데이터로 비교

예를 들어 숫자 1, 2, 2, 3을 저장하면:

a = [1, 2, 2, 3]       # list
b = (1, 2, 2, 3)       # tuple
c = {1, 2, 2, 3}       # set
d = {"a": 1, "b": 2}   # dict

결과 느낌은 이렇게 다릅니다.

list → 순서 있고 중복 허용

tuple → 순서 있고 중복 허용, 수정 불가

set → 중복 제거됨

dict → 키를 이용해 값 관리



---

4. 핵심 차이 예제

수정 가능 여부

a = [1, 2, 3]
a[0] = 100
print(a)   # [100, 2, 3]

b = (1, 2, 3)
# b[0] = 100   # 오류


---

중복 허용 여부

a = [1, 1, 2]
b = (1, 1, 2)
c = {1, 1, 2}

print(a)   # [1, 1, 2]
print(b)   # (1, 1, 2)
print(c)   # {1, 2}


---

접근 방식

a = [10, 20, 30]
b = (10, 20, 30)
d = {"x": 10, "y": 20}

print(a[1])      # 20
print(b[1])      # 20
print(d["x"])    # 10

set은 인덱스 접근이 안 됩니다.

c = {10, 20, 30}
# print(c[0])   # 오류

5. 실무적으로 쉽게 구분하는 방법

list

“줄 세워 놓은 값들”

fruits = ["apple", "banana", "orange"]

tuple

“바뀌면 안 되는 값 묶음”

coordinate = (37.5, 127.0)

set

“중복 없는 값 모음”

tags = {"python", "ai", "python", "ml"}
# {'python', 'ai', 'ml'}

dict

“이름표가 붙은 값들”

user = {"id": "abc123", "name": "Kim"}


---

6. 어떤 상황에 무엇을 쓰면 좋은가

순서 있는 데이터, 수정 자주 함 → list

고정된 데이터 묶음 → tuple

중복 제거, 집합 연산 → set

항목명과 값을 짝지어 저장 → dict



---

7. 매우 중요하게 기억할 한 줄

list = 변경 가능한 순서 있는 목록

tuple = 변경 불가능한 순서 있는 목록

set = 중복 없는 값의 집합

dict = key-value 사전형 자료구조
